Documentation Protips

  • If you ask a question, write the question and the answer in your notes
  • What is the issue? (get detailed info, error messages, screenshots, symptoms)
  • Where is it occuring? (get building, room, etc)
  • When did the problem start? (get the time this started, an hour ago, last week, past 5 months, etc)
  • Is anyone else affected? (this helps you isolate the issue to one user or one workstation, or helps you launch a higher priority ticket)
  • Have you tried turning it off and on again? (no joke, if it applies, do it)
  • Did you REALLY gather all the info you can?

Command Line

  • command - result
  • getmac - for MAC address
  • msinfo32 - lots of info, including if 64bit or 32bit
  • hostname - computer name
  • shutdown /r /t 3 or shutdown -r restarts computer
  • nslookup - shows IP address of my server
  • tracert [ip] - shows computer name if you only have an ip and shows the path a packet of info takes from your machine to the other, can be used for troubleshooting
  • nbtstat - also shows computer name (nbtstat -a computername)
  • echo %logonserver% - shows what server the computer is connected to, to see if it is connected to any server.
  • ping -t computer name - continuous ping. stop w/ ctrl+c
  • wmic /node:"305OSS-8391LFQ" computersystem get username - (computer name in the quotes) gets the edipi of the user currently logged into a computer
  • [command, i.e. getmac] /s computer name - from RAT server to get info on a different computer w/in same server.
  • BACKDOOR to computer: \\computer name\c$
